# 情感突变检测算法文献综述报告

## 一、研究背景

情感突变检测是舆情监测系统的核心功能之一，旨在实时识别公众情感状态的剧烈变化，为预警决策提供技术支撑。本报告基于近5年学术文献调研，系统分析情感突变检测的主流算法方案。

---

## 二、主流算法方案分析

### 2.1 CUSUM 累积和算法

**原理**：通过计算累积偏差检测均值偏移，当累积和超过阈值时判定为突变点。

**公式**：
```
S_t = max(0, S_{t-1} + (x_t - μ) - k)
```
其中 `k` 为参考值，通常设为 `k = δ/2`（δ为预期检测的最小偏移量）。

**优点**：
- 在线实时检测，延迟小
- 对小幅偏移敏感
- 计算复杂度低 O(n)

**缺点**：
- 需要预设参考值和阈值
- 对非平稳序列适应性较差
- 单变量检测

**适用场景**：情感分数均值漂移检测

---

### 2.2 BOCPD 贝叶斯在线变点检测

**原理**：基于贝叶斯推断，通过维护后验分布实时估计"最后一个变点发生时间"。

**核心公式**：
```
P(r_t | x_{1:t}) ∝ P(x_t | r_t, x_{1:t-1}) P(r_t | r_{t-1})
```
其中 `r_t` 为自最后一个变点以来的时间长度。

**优点**：
- 理论基础扎实，可解释性强
- 自动适应数据分布变化
- 可输出变点概率

**缺点**：
- 计算复杂度较高
- 需要指定先验分布
- 对超参数敏感

**适用场景**：流式数据处理，实时舆情监测

---

### 2.3 PELT 精确线性时间算法

**原理**：基于动态规划，通过剪枝策略在 O(n) 时间内找到所有变点。

**目标函数**：
```
min_{τ_1,...,τ_K} ∑_{k=0}^{K} C(x_{τ_k+1:τ_{k+1}}) + βK
```
其中 `C()` 为代价函数，`β` 为惩罚项。

**优点**：
- 精确求解，无近似误差
- 线性时间复杂度
- 可处理多变量

**缺点**：
- 离线算法，不适合实时场景
- 需要指定惩罚参数
- 对异常值敏感

**适用场景**：历史数据分析，批量处理

---

### 2.4 Z-score 统计方法

**原理**：计算当前值与历史均值的标准化距离，超过阈值则判定为异常。

**公式**：
```
Z = (x - μ) / σ
```
当 `|Z| > 3` 时判定为异常（3σ原则）。

**优点**：
- 简单直观，易于实现
- 计算效率高
- 无需训练

**缺点**：
- 假设数据服从正态分布
- 对缓慢漂移不敏感
- 需要足够历史数据

**适用场景**：快速异常检测，初步筛查

---

### 2.5 滑动窗口统计方法

**原理**：在滑动窗口内计算统计量（均值、标准差、分位数），比较相邻窗口差异。

**检测指标**：
- 均值差异：`|μ_1 - μ_2| > threshold`
- 方差比率：`σ_1² / σ_2²` 显著偏离1
- 分布距离：KL散度、JS散度

**优点**：
- 灵活可配置
- 可捕捉多种变化模式
- 适合时间序列

**缺点**：
- 窗口大小选择困难
- 边界效应
- 对噪声敏感

**适用场景**：情感趋势分析，周期性检测

---

### 2.6 SPC 统计过程控制

**原理**：使用控制图（Shewhart图、EWMA图）监控过程稳定性。

**控制限**：
```
UCL = μ + 3σ
LCL = μ - 3σ
```

**优点**：
- 工业验证成熟
- 可视化直观
- 区分普通波动和特殊波动

**缺点**：
- 假设过程稳定
- 对相关数据不适用
- 需要历史基线

**适用场景**：舆情稳定性监控

## 四、推荐算法方案

基于项目需求（实时舆情监测、情感突变预警），推荐以下算法组合：

### 主推方案：CUSUM + 滑动窗口

**理由**：
1. CUSUM 对小幅偏移敏感，适合检测渐进式情感变化
2. 滑动窗口灵活可配置，可捕捉多种变化模式
3. 两者结合可兼顾实时性和准确性
4. 计算复杂度低，适合大规模数据处理

### 备选方案：BOCPD

**理由**：
1. 理论基础扎实，可输出变点概率
2. 自动适应数据分布变化
3. 适合对准确性要求极高的场景

### 辅助方案：Z-score

**理由**：
1. 简单快速，适合初步筛查
2. 可作为其他算法的前置过滤
3. 易于理解和维护

## 五、实施建议

### 5.1 算法实现优先级

1. **Phase 1**：实现 Z-score + 滑动窗口（快速上线）
2. **Phase 2**：实现 CUSUM（提升准确性）
3. **Phase 3**：实现 BOCPD（高级功能）

### 5.2 参数调优建议

| 参数 | 建议值 | 说明 |
|------|--------|------|
| 滑动窗口大小 | 30-60分钟 | 根据数据频率调整 |
| Z-score 阈值 | 2.5-3.0 | 平衡灵敏度和误报率 |
| CUSUM 参考值 k | δ/2 | δ为预期最小偏移量 |
| CUSUM 阈值 h | 5σ | σ为标准差 |

### 5.3 评估指标

- **检测延迟**：变点发生到检测的时间差
- **准确率**：正确检测的变点比例
- **误报率**：错误检测的变点比例
- **F1分数**：准确率和召回率的调和平均
